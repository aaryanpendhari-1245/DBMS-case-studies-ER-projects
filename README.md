# DBMS-case-studies-ER-projects
Comprehensive database design projects featuring ER diagrams, relational schemas, and SQL queries for real-world systems like University, Hospital, and E-Commerce.

DBMS Case Studies – ER Modeling & Database Design
📌 Overview
This repository contains three comprehensive database case studies demonstrating Entity-Relationship (ER) modeling, relational schema design, and SQL implementation.

The goal of this project is to showcase practical understanding of Database Management Systems (DBMS) concepts through real-world scenarios.

📁 Case Studies Included
🎓 1. University Database
Student, Course, Instructor, Department management

Weak entities (SECTION)

Many-to-Many relationships (ENROLLMENT)

Multi-valued attributes (Email, Phone)

🏥 2. Hospital Database
Patient, Doctor, Nurse, Ward system

Associative entities (TREATMENT, ADMISSION)

Weak entities (EMERGENCY_CONTACT, LAB_TEST)

Complex relationships for prescriptions and treatments

🛒 3. E-Commerce Database
Customer, Product, Order, Vendor system

Self-referencing relationships (CATEGORY hierarchy)

Weak entities (ORDER_ITEM, ADDRESS)

Advanced features like Reviews, Payments, Cart system

🧠 Concepts Covered
Entity-Relationship (ER) Diagrams

Strong & Weak Entities

Cardinality & Participation Constraints

Normalization (1NF, 2NF, 3NF, BCNF)

Relational Schema Design

Primary Keys & Foreign Keys

Many-to-Many Relationships (Junction Tables)

Multi-valued & Derived Attributes

🛠️ Tools & Technologies
Draw.io / StarUML (for ER diagrams)

SQL (MySQL / PostgreSQL)

GitHub (version control & documentation)
